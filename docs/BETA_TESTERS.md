## Beta Onboarding Flow

### What Testers Get Access To

1. **Early Access CLI** - Pre-release version of Clawpipe
2. **All Scenario Packs** - Including premium packs (Basic Sanity, Chaos Monkey, etc.)
3. **Direct Feedback Channel** - Private Discord channel or dedicated Slack
4. **Credit on Launch** - Recognition in launch blog post (optional)

### How to Run First Test

```
1. Install: npm install -g clawpipe@beta
2. Auth: clawpipe login (or use demo mode)
3. Run: clawpipe run --scenario basic-sanity
4. View: clawpipe report --last
5. Share: clawpipe share (generates public URL)
```

### Feedback Collection

| Method | When | Purpose |
|--------|------|---------|
| Daily check-in | First week | Blockers, setup issues |
| Discord thread | Ongoing | Quick questions, wins |
| Feedback form | After 5 runs | Structured NPS, feature requests |
| Exit interview | Beta end | Final impressions, pricing feedback |

**Feedback Form Fields:**
- Net Promoter Score (0-10)
- What worked well
- What was confusing
- Would you pay for Pro? How much?
- Any features you needed that were missing
